## ✨ Features

- **Desktop Application**: Fast, native experience with a modern UI and responsive dark mode style.

- **AI-Powered**: Uses efficient Ollama vision models (like `llama3.2-vision`) to analyze images locally.

- **Smart Renaming**: Automatically generates descriptive filenames based on the image content (e.g., `sunset_mountains_lake.jpg`).

- **Auto-Organization**: Move and organize images into dedicated folders based on semantic tags generated by AI.

- **Privacy Focused**: All processing happens entirely locally on your machine—no cloud uploads or online APIs used.

- **Format Support**: Supports standard image formats such as JPG/JPEG, PNG, GIF, BMP, and WebP.

## 📖 Prerequisites Setup

1. **Install Ollama**: Download from [ollama.ai](https://ollama.ai)
2. **Pull a vision model**:

   ```bash
   ollama pull llama3.2-vision
   ```

   *(Note: You can use other vision-capable models as well)*

## 🚀 Usage

1. **Start Ollama**: Ensure your local Ollama instance is running (default `http://localhost:11434`).
2. **Select Model**: Upon launching PicTag, select your vision model from the dropdown.
3. **Choose Source**: Browse for or paste the path to your folder of images.
4. **Scan**: Click "Scan" to load the images.
5. **Generate Names**: Let the AI analyze each image and propose a new filename. You can review the proposed changes in the interface.
6. **Apply Changes**: Confirm to safely rename the files on disk.
7. **Organize**: Use the "Organize by Tags" section to automatically move files into thematic subfolders based on common keywords.

## 🛠️ Development

- **Backend**: Rust (Tauri)
- **Frontend**: HTML/CSS/JS (Vanilla, Single File)
- **AI**: Ollama (Local API)

1. **Install Dependencies**: Ensure you have Rust/Cargo, Node.js, and Tauri prerequisites (WebView2 on Windows).
2. **Install Frontend Packages**:

   ```bash
   npm install
   ```

3. **Run in Development Mode**:

   ```bash
   npm start
   ```

   This runs the frontend dev server and the Tauri backend concurrently.

4. **Build for Production**:

   ```bash
   cargo tauri build
   ```

   The executable will be generated inside `src-tauri/target/release/` (or `target/release/` depending on configuration).
